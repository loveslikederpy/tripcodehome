project #1: elastatripper

The Elastic Tripcode Generator Kernel.

The aim of this project is to create a highly flexible tripcoder that will use any or all processors available to it--including cloud-based ones--to generate tripcodes as quickly as possible from any platform.

project #2: tripperman

The Elastic Tripcode Manager

The aim of this second project is primarily to create a powerful and entertaining GUI for Tripcode@Home management. Specifying the desired tripcodes and to what accuracy (1337, for example) are the basics here. Neworking tools for connecting with distributed local and remote Cloudtripper servers should be provided. Additionally, entertaining and user-devisable skins/animations such as a "Derpy gets ALL THE MUFFINS!?" theme are intended. And eventually, a builit-in wallet enabling transactions using the Chickunbits cryptocurrency system is envisioned for inclusion into the project.

project #3: cloudtripper

The Elastic Tripcode Cloud Service

The aim of this third project is to enable distributed tripcode generation via typical cloud technologies. The desired outcome is for various communities to band together to provide each other distributed generation services and sharing. Push/Pull requests, usage arbitration, and distributed store/forward mechanisms should be provided. Community-browsable general repositories of generated results should also be provided for sharing. The Tripperman manager will check these repos first for possible immediate results back for any particular user tripcode request.

overall project plans: Tripcode@Home

So basically, the overall system project plan is to deliver 3 fundamental features:
1) A portable compute kernel that flexibly generates tripcodes.
2) Cool GUIs for this kernel and system that work on all platforms.
3) A community-driven cloud service to share tripcode generation with others.
